Octopress on a Raspberry Pi

Sitting in the corner of my study is a Raspberry Pi, attached to my home network through a wireless connection provided by an Edimax EW-7811UN 150Mbps Wireless Nano USB Adapter1.

I absolutely adore the little device and it’s versatility means I’ve been able to mess around and play to my hearts content over the last few months. Amongst the vast array of software, hacks and peripherals you can add to the device, to date I’ve been able to;

  • Turn it in to an AirPlay Receiver using a Dynamode USB Sound Card.
  • Create an L2TP VPN server for my iOS devices
  • Create a Transmission torrent server
  • Setup a DLNA Server for streaming movies to my PS3
  • Installed Octopress for this very blog.

Self Hosting

This week I’ve also moved hosting of this blog across to the Pi and away from Github Pages. I’m sure the Pi will cope quite admirably with the trivial amount of traffic this blog receives on a daily basis and with extra protection provided by CloudFlare I don’t have any real concerns over performance, in fact it should run extremely well.

I decided upon Nginx as it’s web server, being fairly lightweight and relatively trivial to configure with Octopress so it’s now a simple case of watching the Pi’s performance closely in the coming weeks.

Dynamic DNS is the only piece of this self-hosting puzzle not satisfactorily resolved as yet2. I’m currently using a modified version of ddclient with a CloudFlare hack which seems to work in verbose/debug mode, although time will tell if that really is the case on an automated basis once my external IP changes in the coming days/weeks.

Workflow

Octopress and a Raspberry Pi fit perfectly together, I can be completely mobile yet still write content (and include photographs) and publish them to this blog with ease using a combination of the following software and workflow;

  1. Create a new post content using iA Writer before saving to Dropbox.3
  2. SSH to the Pi using a brilliant bit of software called Prompt.
  3. Download the newly created post to the RPi’s Octopress _posts folder using Dropbox Uploader.
  4. Finally, using Prompt again, perform a ‘rake generate’ to update the content on the Pi’s NGinx web server and present it to the world.

It’s also worth mentioning another great iOS App called Textastic which enables me to also modify my Octopress source code and theme whilst mobile too.

This combination of Apps and mobile devices is actually making my blogging a pleasure.

  1. I highly recommend this USB Adaptor if you want to go wireless with your Pi. ↩

  2. Why don’t British Telecom give the option of a static IP with their Infinity product? ↩

  3. At this point if I was to include a photo in my post I’d save this to the Public folder on my Dropbox account. ↩

Dynamic DNS Updater for CloudFlare

Over the weekend I decided to move hosting of this blog from Github Pages to the Raspberry Pi that resides in the bedroom of my home1. The site has always been protected by the impressive CloudFlare but that protection is even more critical now I’ve made that change, it does however present a particular challenge around dynamically assigned IP addresses that ISP’s dish out on consumer broadband and fibre networks.

CloudFlare needs to be kept informed of the currently assigned IP address that the Pi is sitting behind and although they do offer an API and a dyndns client, it’s a little troublesome to get working so I decided the best approach would be a simple bash script.

The script runs perfectly on a Raspberry Pi and all it needs is a simple alteration to include your own domain, CloudFlare Username and API key, then add it to a cron job and it will keep your CloudFlare DNS up to date with the currently assigned dynamic IP at an interval of your choosing2.

Any support, issues, comments or feedback are always welcome by getting in touch on Twitter.

  2. I currently have mine set to update CloudFlare every 55 minutes. ↩

An Octopress Flickr Set Plugin

While I was in the process of setting up and theming this site I envisaged writing a post that documented the migration of my Instagram photos to Flickr1, the idea being that I would then embed the newly migrated Flickr photo set in said post on this Octopress powered blog.

I wasn’t entirely sure whether I’d have to write my own plugin to do so or if I’d be lucky enough to find one already out there and I was thankfully saved the work after coming across a great little Octopress plugin by tsmango that did almost everything I was after, however I felt it could be improved upon so decided to fork the project and make some changes of my own.

  • Firstly it returned all photos in a set, the Flickr API can return up to 500 photos at a time and that wouldn’t display too well in a blog post, so I’ve added a new parameter which allows you to configure the amount of photos that are returned by changing the newly created ‘per_page:’ setting in the _config.yml to a number of your choice.

  • Secondly, clicking a photo thumbnail would take you directly to the source image while I personally thought it more sensible if it took you to that individual photos full Flickr page instead, another parameter is required for this called ‘user:’ in which you enter your own flickr userID.
